认识职场

发表于2025-12-05
|总字数:5k|阅读时长:15分钟

认识职场

找工作

简历

基本信息

  1. 姓名、电话、邮箱。
  2. 其他个人信息一般不必需，除非岗位有特殊要求。
  3. 照片按投递岗位要求提供。

教育经历

  1. 院校（985/双一流标注）、专业（双一流标注）、修读时间必需，且按照时间由近及远排布。
  2. 成绩和排名如果比较好，可以提供。
  3. 修读课程按投递岗位要求和课程成绩优劣酌情提供。
  4. 在读期间如有代表性奖项或荣誉可以简略提供。

工作/实习经历

  1. 公司名称、职位、工作/实习时间必需，且按照时间由近及远排布。
  2. 每份工作/实习的经历描述，结合投递公司和岗位要求撰写相关内容，可以按照STAR原则的结构，切忌面面俱到或过于简略。

注：STAR原则，即Situation背景、Task任务、Action行动、Result产出。

科研经历/项目经历/校园经历等其他经历

  1. 相关科研、项目、组织/…名称、职能、经历时间必需，各自板块按照优势和岗位要求进行排序。
  2. 各个板块内部如有多项，时间由近及远排布。
  3. 各个经历描述，结合投递公司和岗位要求撰写相关内容，可以按照STAR原则的结构，切忌面面俱到或过于简略。

例如：

  • 参数调优：基础参数调优以及基于火焰图的精准调优，实现 tpmC（每分钟订单处理量）提升 40% 至 22,503；通过压缩方法和多套参数配置，实现导入速度提升 50%，最高支持 18,000+ 并发连接

  • 硬件加速：基于 Intel® DSA 实现内存操作优化，高并发场景性能提升超 10%。

站在面试官的角度，他肯定不会期望一个实习/校招的候选人有什么超强超全面的技术理解，但只要有一个点你能够深入的讲解（就比如缺页异常，结合自己在minios中的实现以及现实场景的八股考虑进行说明，就是比较亮眼的点），那么就说明出你是有思考过程的。

其他内容

  1. 根据个人特质和岗位要求，可以有资格证书、技能、兴趣爱好、个人评价等内容，但最好和职位相关或是你个人亮点。
  2. 个人评价一般不写，容易有凑字数嫌疑，如果一定要写建议转换为个人优势，简略写。

简历规范

  1. 篇幅：实习/校招简历严格控制在一页A4，社招简历严格控制在1-2页A4。
  2. 版面：干净整洁，不建议用网络模板（特别是花哨的模板），最好自己定制。
  3. 表述：切忌口语化、空洞化，用简练、准确的语言阐述经历。
  4. 细节：不能有错别字、标点符号错误、语病、序号错误、没对齐、字体和字号杂乱。
  5. 文件：如果提交简历电子版，建议PDF版本。

各类经历撰写

  1. 写入简历的经历务必都进行描述，否则就不要写；
  2. 务必要体现你在其中承担的工作（动宾）和最后的产出（如果能量化请务必采用数据），切忌经历写得大而全；
  3. 务必要和求职岗位要求匹配，无关经历不仅不会给简历加分，反而会影响HR和面试官发现你的亮点。
  4. 可以用加粗标黑突出亮点，但加粗标黑的部分一定要是基于某种逻辑的亮点，不要什么都标，什么都是重点最后就都不是重点。
  5. 不同公司/职位，务必根据JD（Job Description 职位描述），定制简历。

职位

外企

求职渠道

  1. LinkedIn＞NiceShark＞猎聘=51job≥智联=boss、神仙外企、菁客…

  2. 企业公众号或者官网

  3. 小红书、知乎、脉脉等搜索直接岗位+地区

  4. 实习：应届生求职、实习僧

  5. 当地线下招聘会、行业展会、高校双选会

  6. 内推:牛客、脉脉、亲朋好友、校友会..建议优先投递带有“紧急招聘”或“urgent hire”标签的岗位。

  7. 通过工具如 Hunter.io 获取目标公司员工的邮箱，重点联系入职6-12个月的初级员工。他们通常刚过试用期，有推荐权限，且推荐奖金对其有一定吸引力。

  8. 在平台搜索“公司名+china+hiring+manager”，有时能直接对接招聘负责人，提高效率。

  9. 在工作日统一投递，简历要根据岗位去变化

在 LinkedIn 上搜索目标岗位时，可使用“过去24小时”筛选。如果想获取更实时的信息，可复制网址栏中带有“86400”的数字参数，将其改为“3600”，即可查看最近一小时内更新的职位。

在平台可直接使用“外资”“欧美企业”“500强”“WFOE”（外商独资企业）等标签进行筛选，或输入英文职位关键词（如“Marketing Manager”）加地点（如“Shanghai”）精准搜索。

判断公司是否为真正的外企，可使用「天眼查」或「企查查」查看股东信息。若大股东为境外注册公司（例如在巴黎），通常为真实外企。需注意部分企业名称中带“国际”但实为本土公司。

使用 Glassdoor 等平台查看公司评价。若出现“加班多”“流程混乱”等内容，需谨慎评估该外企的实际情况。

项目

深挖细节，展现思考

不要直接在简历里写课程项目，起码进行一些简单包装，哪怕问到了也最好说是额外的实现。从全局的角度理解数据库，操作系统的架构，深化自己在细节上的处理以及自己的理解。

量化

内容：

  • 数学/统计、概率、优化、时间序列、机器学习基础
  • 编程、工程能力、代码质量
  • 作品集：可复现的研究、回测框架、因子研究、风险控制、交易成本、稳健性检验
  • 面试/笔试/机考/白板/brain teaser/概率题/系统设计（视机构而定）

展示：（相当于 quant 的“路演替代品”）：

  • GitHub：完整的研究流程（数据→因子→回测→检验→风险→复盘）
  • 研究报告：像 sell-side/买方研究一样写清楚假设、结果、失效场景
  • 比赛：Kaggle/天池/数模/量化相关赛（注意合规和可解释性）

行业分析

内容：

  • 行业研究样本（2-3篇深度）
  • 财务建模 + 估值（尤其卖方研究）
  • 写作 + PPT表达

路径：

  • 参加券商/研究所的公开活动、研报写作赛、行业论坛报告展示
  • 在公开平台发布高质量研究（注意合规：不要泄露非公开信息/不要引导交易）

券商 / 投行

后台岗位：

  • 交易系统/柜台/OMS/EMS/撮合相关（低延迟、稳定性、风控、并发）

  • 清算/结算/资金/风控系统（强一致性、账务、对账、权限、审计）

  • 数据平台/实时计算/报表（ETL、流处理、指标体系、数据质量）

  • 量化/研究/投研支持平台（但这偏“技术+业务”）

  • 信息安全/基础设施/运维开发（合规、安全、灾备）

内容：

  • 工程习惯：

    • 日志、异常处理、权限、审计、可回溯

    • 数据一致性、对账思维

    • 接口设计、测试、文档

  • 金融业务理解：

    • 订单（委托）→ 撮合 → 成交 → 清算/结算 → 资金/证券交收

    • 风控（限额、保证金、穿仓、持仓、风控规则）

    • 对账（系统间、日终、差异处理）

展示：

  • 交易案例拆解（找真实公告做复盘）
  • 建模（3表、DCF、可比公司、可比交易）
  • PPT能力（结构化表达：项目亮点/风险/对标/估值/卖点）

做一个“统一可迁移”的作品包

  1. 选一个你愿意长期跟的行业（比如新能源/半导体/互联网/医药/消费）
  2. 写一篇“深度报告”（行业框架 + 龙头公司财务拆解 + 风险点）
  3. 配一个模型（Excel/代码：估值或回测）
  4. 做 6 页 PPT + 3 分钟讲稿

面试

面经

牛客、脉脉、98

装逼

  • 你已经是这方面的专家了。

    • 我确实是这方面的专家，在xxx的背景下（这个很重要），xx文献说了xx，结合我做的课题，我认为xxx。其实只要比对方多看了几篇文献，甚至直接用大模型帮忙看几篇，记忆其中的一些关键点就好了，最重要的还是自信和流利的表达。
    • 能说出“你是这领域的专家”的导师，大概率也是个草包。他当然也有可能肚子里有真货，那样子也挺好的，至少能真的指导一下。所以这件事，要不然就是能唬住对方，要不然也能得到一些有用的建议，都是利好的。

  • 面试：谈背景、谈理想、谈贡献，信息差来源——你做过他没做过，在这个细分的领域，你比他多懂一些。如果他真的做过，那也是好事，说明你这段经历是垂直的，你可以通过这一番交流，弥补自己的认知，甚至能套到一些对方的见解，学会了就是自己的。

    • 先从背景讲起：这是个什么样的事情，要尽可能通俗易懂，想象在听学术报告。
    • 你为什么要做这件事情？讲清楚这件事情在整个链路中的作用，再辅以人力/物力/资源角度的个人见解。例如强化学习：业务场景是怎么怎么样，面对这种业务，微调的劣势在哪里，强化学习优势在哪里，结合八股讲，强化学习它更加注重探索，说不定能发现人类发现不了的东西，举一个例子alpha go，也可以说微调要标数据，人力成本比较大，再举deepseek的例子，用强化学习去激发思维链，比人类标注CoT更省钱。回到实际，两个都有探索，从效果角度来看，强化学习更好一些，微调有xxx的问题，这里可以结合badcase分析，表明真的做过了。为啥用微调不用强化学习呢？背点微调的优势，微调在公司里已经是平台化的一个东西了，更加符合即插即用、高效探索，有利于项目快速落地。

入职以后

行业假设： 对某个行业的驱动因素、竞争格局、关键成功要素和未来趋势提出的初步猜想。

  • 举例：分析“预制菜行业”，一个初始假设可能是：“这个行业增长的核心驱动因素是都市单身青年和双职工家庭对便捷餐饮的需求提升，而非单纯的口味。” 之后的所有访谈和数据收集，都会围绕验证或推翻这个假设展开。

Delivery Deck： 向客户正式交付的、包含最终结论、详细分析和行动建议的演示文件合集。普通PPT可能是信息罗列，而Delivery Deck是一个有观点、有逻辑、旨在驱动决策的商业论证。

  • 结论驱动：首页就是核心结论（The Answer）。
  • 逻辑严密：遵循“背景-问题-分析-结论-建议”的金字塔结构。
  • 故事线清晰：每一页PPT都是整个说服性叙事的一部分。

茴香豆（bushi）PPT的三种称呼😂：

  • Slides / PPT：指代的是单个页面或技术文件，强调其“载体”属性。新手常停留于此，只关注每页是否美观、信息是否填满。
  • Deck：指的是一整套幻灯片，像一个完整的“牌组”。这个词强调其整体性和成套性。当说“把Deck发给我”时，意味着需要一整套有逻辑、能完整表达一个故事的文件。
  • Presentation：它指的不仅是文件本身，更是 “演示”这个动态过程。它包含了演讲者的叙述、与观众的互动、以及最终要达成的说服目标。一个优秀的Presentation，其灵魂是演讲者的叙事，Deck只是辅助工具。

商业思维：一种以创造和获取价值为核心，以逻辑和框架为工具，以驱动决策和行动为导向的思维方式。

  • 本质驱动：不停留于表面现象，通过连续问“为什么”，找到问题的根本原因。
  • 用户是谁：他为什么在乎？什么场景？什么替代方案？
  • 价值导向：始终追问“这能为客户/公司创造什么价值？如何量化？”，你解决的痛点/省钱/省时/增收/增安全感？
  • 怎么赚钱：谁付钱、为什么付、付多少、多久付一次？
  • 怎么持续：获客成本、交付成本、壁垒、复购/口碑/网络效应？
  • 看财报/商业新闻只问一句：它的增长来自“用户变多/付费变高/成本变低/效率更高”哪一个？
  • 结构化：运用金字塔原理、MECE法则等，将复杂问题分解得清晰、完整。
  • 假设验证：先建立猜想，再用数据和事实快速验证，高效推进。
  • 效益成本意识：任何建议都权衡投入产出比和风险。

Key Success Factors(KSF)：，即关键成功要素。指在某个特定行业中，企业要获得成功必须做好的少数几件核心事情。

  • 护城河：这往往是业内公司最核心的竞争机密和优势所在。例如，对于零售业，KSF可能是“供应链成本与效率”；对于 SaaS 软件，可能是“客户留存率”或“净推荐值”。
  • 专家价值：资深行业专家之所以有价值，正是因为他们对KSF有深刻、直观的理解。在访谈中直接问“您行业的KSF是什么？”会显得非常外行且冒犯，因为这是在索取对方最核心的知识财富。
  • 正确的打开方式：应该通过自己的初步研究形成对KSF的假设，然后在访谈中用具体问题去印证或修正。例如，不问“KSF是什么？”，而是问：“如果一个新玩家想进入这个市场，您认为它在头两年最需要克服的三个挑战是什么？” 后者更具体，也更能引出真知灼见。

礼仪谈吐：

  • 结论先行：我是谁 + 我做什么 + 我想要什么
  • 两点证据：用 2 个事实支撑（项目、数据、案例）
  • 给对方一个台阶：提供选择/下一步（“你看A或B哪个更合适？”）

30秒自我介绍模板

我是___，主要做___（面向谁/解决什么）。过去___（一个成绩/案例）。我现在在关注___，如果你也有这类需求/资源，我们可以约个时间具体聊。

  • 不抢话：先复述对方重点再回应
  • 不输出价值观：少说“应该”，多说“我观察到/我担心/我建议”
  • 不轻易提要求：先给选项、给时间成本预估、给退出机制

职场新人成长

心态：

  1. 主动进行“逆向工程”：拿到模板后，不要只填空。反复问自己：

    • 这一页在整个故事里扮演什么角色？（是定义问题，还是分析原因，还是提出方案？）
    • 页面之间的逻辑是怎么衔接的？
    • 如果换掉这个数据或图表，会影响整个论证吗？
    • 这个模板结构，反映了作者怎样的思考路径？

  2. 以“学徒”心态进行安全提问：

    • 不问：“这个怎么做？”（太宽泛，把责任推给他人）
    • 而是问：“关于这部分分析，我参考了之前的XX案例，尝试用A思路来推导，您看这个方向对吗？”（展示你的努力和思考，寻求方向性指点，对方更容易开口）
    • 或者：“为了更好理解这个模块的意图，能分享一下这份分析最终影响了客户哪个层面的决策吗？”（从商业价值反推工作意义）

  3. 理解并尊重职场“潜规则”：

    • 保密要求与知识传承的平衡：老员工不教，有时并非自私，而是受制于严格的保密协议，或担心信息被误用。他们给出模板，可能已是在合规范围内能提供的最大帮助。
    • 成年人的学习模式：职场不是学校，系统性培训是奢侈品。高手通常只会在看到你主动思考、遇到具体关卡时，才愿意给出“点睛”之笔。

路演

个人或团队把自己的项目/研究/商业洞察做成“可讲述的成果”，在企业、咨询/PE相关社群、比赛、校招活动里对“评委/企业代表/前辈”展示。目的：拿到面试、内推、实习、offer、导师资源。

普通人怎么获得路演机会？

  1. 从“公开入口”的路演/展示场开始

    • 创业/创新比赛与本地孵化器/产业园Demo Day

      • 各地科创部门、高校创业中心、孵化器、加速器、产业园，常年有比赛/训练营/路演日
      • 不一定要真开公司，可以是“项目提案 + MVP/调研报告”

    • 黑客松 / 产品增长赛 / 数据分析赛

      • 很多企业或社区会办

    • 行业峰会的“青年论坛/学生论坛/Poster展示/Workshop”

  2. 加入团队

    • 加入一个正在融资/在加速器里的团队

      • 你在里面做：行业研究、商业计划书BP、数据分析、增长、产品、运营、BD
      • 很多团队会带核心成员一起参加 Demo Day/路演，你自然能进场

    • 怎么找？

      • 创业社群、孵化器招募栏、校友群、比赛队伍、开源社区、产品社区

    • 你的优势是什么？

      • 你不靠资源，靠“能干活+能交付”：能做一份像样的市场研究/竞品/用户访谈/财务模型/增长方案，就会被需要

  3. 做“可被转发的展示材料”

    • 一页纸（One-pager）：你是谁 + 你解决什么问题 + 你做过什么证明 + 你想要什么对接

    • 3分钟路演稿（可口播）：问题-方案-证据-需求（别讲宏大叙事）

    • 作品证据：项目链接 / 研究报告PDF / 原型 / 数据看板 / 公开视频

索取

  • 你希望对方给你什么？
    • 求职：内推/面试/实习/项目合作
    • 项目：试点客户/导师/投资/渠道
  • 你给对方什么？
    • 结果（数据/案例）、时间（能干活）、稀缺信息（研究/洞察/资源整合）
  • 换到对方的下一步：加微信 + 约10分钟线上聊 + 发一页纸
  • 跟进：发材料 + 复述对方关注点 + 给出一个“可执行的下一步”

行动

  • 做一份一页纸（个人/项目都行）
  • 准备一个3分钟口播 + 6页PPT（问题、方案、证据、案例、你能做啥、你要啥）
  • 选择一个赛道：例如消费/AI应用/跨境/硬科技/教育/金融等，别太泛
  • 参加公开比赛/训练营/峰会青年论坛
  • 争取进入一个正在做事的团队（兼职/实习/志愿）
  • 用一次路演产出：公开视频/复盘文章/作品链接 → 换下一次邀请
